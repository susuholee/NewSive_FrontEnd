## 프로젝트 소개

**NewSive**는 실시간으로 발생하는 뉴스와 정보성 이벤트를 빠르게 전달하고,  
알림 및 커뮤니티 기능을 통해 사용자 간 즉각적인 반응과 소통을 지원하는  
**실시간 정보 중심 플랫폼**입니다.

사용자는 최신 뉴스 데이터를 기반으로  
친구 및 커뮤니티와 의견을 나누며 **실시간 반응과 참여가 가능한 경험**을 제공합니다.

---

## 개발 목적

최근 사회적 이슈와 뉴스는 실시간으로 빠르게 발생하지만,  
젊은 세대는 주로 인스타그램, 유튜브와 같은 **대중적인 플랫폼**을 통해 정보를 접하는 경우가 많습니다.  
이러한 플랫폼은 접근성은 높지만,  
개인의 관심사와 무관하게 파편화된 정보 소비로 이어지기 쉽다고 느꼈습니다.

NewSive는 이러한 문제의식에서 출발하여,  
**젊은 사용자들이 실시간으로 발생하는 사회적 이슈를 보다 쉽게 접하고**,  
뉴스를 중심으로 한 **커뮤니티 기반 소통**을 통해  
사회 전반에 대한 관심과 참여를 자연스럽게 유도하는 것을 목표로 개발한 서비스입니다.

## 아키텍처 설계

- **Frontend**
  - Next.js (App Router)
  - Zustand 기반 전역 상태 관리
  - React Query 기반 서버 상태 관리
- **Backend**
  - NestJS 기반 REST API
  - JWT 인증
  - WebSocket(Socket.IO) 실시간 통신
- **Infra**
  - AWS S3 / CloudFront
  - AWS EC2
  - Route 53 / ACM

---

## 기술 스택

- **Framework**: Next.js (App Router), NestJS
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **State Management**: Zustand
- **Server State**: TanStack React Query
- **HTTP Client**: Axios
- **ORM**: Prisma

## 이슈 상황 및 해결

| 이슈 | 해결 |
|------|------|
| WebSocket 연결 이후 채팅 이벤트 응답 미수신 | HTTP 인증(JWT Guard)이 WebSocket에 자동 적용되지 않는 문제를 인지하고, Socket 전용 JWT 인증 로직을 구현하여 정상적인 이벤트 흐름 처리 |
| 친구 추가 시 데이터 불일치 발생 | 친구 관계 생성 시 양방향 관계(User–Friend)와 알림 생성 로직을 하나의 트랜잭션으로 묶어 처리하여 데이터 무결성 보장 |

## 느낀 점

NewSive는 기획부터 개발, 배포까지 전 과정을 혼자 진행한 프로젝트였습니다.  처음에는 실시간 기능을 직접 구현해보고 싶다는 생각으로 시작했지만,  
막상 진행하면서 기능을 구현하면서 서비스 구조를 어떻게 잡아야 하는지가 더 중요하다는 걸 느끼게 되었습니다. 특히 WebSocket을 염두에 두고 구조를 설계하면서 지금 당장 완성하지 않더라도, 나중에 확장할 수 있는 방향으로 코드를 작성하는 게 얼마나 중요한지 체감할 수 있었습니다. 이번 프로젝트를 통해 화면 구현에만 집중하는 개발이 아니라, 서비스 전체 흐름을 고려하며 개발하는 경험을 할 수 있었고, 앞으로 기능을 추가하거나 개선할 때도 이런 관점으로 접근해야겠다는 생각을 하게 되었습니다.
